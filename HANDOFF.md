# Handoff

## Current Goal

將目前固定值的溫度回傳，改為使用 `sin` 波形模擬溫度變化。

目標行為：

- 溫度範圍為 `-40` 到 `85`
- 每次一次完整的問與答後，溫度都要依照目前波形位置順勢往下一個點變化
- 不要每次重新隨機產生獨立溫度值
- 效果需呈現連續、平滑的升降，而不是跳動式變化

設計方向：

- 維護一個持續累進的 phase 或 step counter
- 每次查詢溫度時推進一次 phase
- 以 `temp = center + amplitude * sin(phase)` 產生溫度
- `center = 22.5`
- `amplitude = 62.5`

補充：

- 若通訊協定只能回整數值，最後需做整數化處理
- 若封包格式不支援直接傳負值，需依實際協定做 signed/offset/多 byte 轉換
